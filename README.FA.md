# سامانه پیامک و وفاداری مشتریان – صفحات داینامیک با Next.js

یک سیستم مدرن و بهینه‌شده برای سئو که برای یک محصول SaaS فرضی ساخته شده است. این محصول به کسب‌وکارهای کوچک کمک می‌کند پیامک ارسال کنند، مشتریان وفادار بسازند و نوبت‌دهی را خودکار کنند.

## 🚀 تکنولوژی‌ها

- **Next.js 16.2.6** (مسیریابی برنامه، توربوپک)
- **React 19.2.4**
- **TypeScript 5** (حالت سخت، بدون `any`)
- **Tailwind CSS 4** + افزونه تایپوگرافی
- **SQLite** با `sql.js` (جاوااسکریپت خالص، بدون کامپایل)
- **Zod** – اعتبارسنجی در زمان اجرا
- **Remark + Rehype** – تبدیل مارک‌داون به HTML در سمت سرور

- ✅ مسیرهای داینامیک: `/[pageType]/[jobType]`
- ✅ رندر در سمت سرور (SSR) – HTML کامل در view-source
- ✅ تولید مسیرهای استاتیک (`generateStaticParams`) برای SSG
- ✅ Feature flag – تغییر بین JSON و SQLite با متغیر محیطی `DATA_SOURCE`
- ✅ آماده برای سئو – متادیتای داینامیک، نقشه سایت، robots.txt
- ✅ محتوای فارسی با راست‌چین
- ✅ طراحی واکنش‌گرا + اسکلت لودینگ + صفحه ۴۰۴ سفارشی
- ✅ پشتیبانی از مارک‌داون (جدول، لیست‌ها، HTML خام) در سمت سرور
- ✅ بدون `any` – ایمنی کامل نوع با Zod و اینترفیس‌ها

## 🛠️ راه‌اندازی

### ۱. کلون و نصب

`bash
git clone ...
npm install
``

### ۲. متغیر محیطی

فایل `.env.local`:

``
DATA_SOURCE=json # یا 'db'
`

### ۳. اجرا با JSON

`bash
npm run dev
`

### ۴. استفاده از SQLite

ابتدا دیتابیس را پر کنید:

`bash
npm run seed
``

سپس در `.env.local` مقدار را به `DATA_SOURCE=db` تغییر دهید و سرور را مجدداً راه‌اندازی کنید.

## 🧪 اسکریپت‌ها

- `npm run dev` – شروع سرور توسعه
- `npm run build` – بیلد نهایی
- `npm run start` – اجرای بیلد
- `npm run seed` – پر کردن SQLite از `data/content.json`

## 📝 نکات معماری

- **لایه انتزاع داده** – اینترفیس `DataSource` امکان تغییر بین JSON و دیتابیس را بدون تغییر در صفحات فراهم می‌کند.
- **الگوی کارخانه** – `getDataSource()` بر اساس `DATA_SOURCE` provider مناسب را برمی‌گرداند.
- **اعتبارسنجی با Zod** – ساختار JSON در زمان اجرا بررسی می‌شود و کوئری‌های دیتابیس اشیاء با تایپ قوی برمی‌گردانند.
- **مارک‌داون سمت سرور** – تبدیل مارک‌داون به HTML در کامپوننت `ServerMarkdown` انجام می‌شود تا HTML نهایی در view-source دیده شود.
- **پشتیبانی از راست‌چین و فارسی** – المان‌های ناوبری، نان‌دون و صفحه اصلی کاملاً فارسی هستند.
- **تولید مسیرهای استاتیک** – `generateStaticParams` تمام مسیرهای شناخته شده را در زمان بیلد می‌سازد.

## 🧠 چرا این کتابخانه‌ها؟

| کتابخانه | کاربرد |
|-----------|--------|
| `remark` | تجزیه‌کننده AST مارک‌داون |
| `remarkGfm` | پشتیبانی از جداول، خط‌خورده و لیست‌های وظیفه |
| `remarkRehype` | پل ارتباطی به AST اچ‌تی‌ام‌ال |
| `rehypeStringify` | تبدیل AST به رشته اچ‌تی‌ام‌ال |
| `rehypeRaw` | اجازه حفظ HTML خام درون مارک‌داون |
| `sql.js` | SQLite خالص جاوااسکریپت (بدون کامپایل، کار در شبکه محدود) |
| `zod` | اعتبارسنجی نوع در زمان اجرا |
